# Review Translation

번역물의 품질을 체계적으로 검수하는 스킬.

## 검수 워크플로우

### 1. 준비
- 원본 파일과 번역 파일을 모두 Read
- 용어집이 있으면 로드
- 같은 프로젝트의 다른 번역 파일이 있으면 용어 사용 패턴 확인

### 2. 구조 검증 (자동)
- 헤딩 수 비교 (원문 vs 번역)
- 코드 블록 수 비교
- 링크 수 비교
- 이미지 참조 보존 확인
- 코드 블록 내 코드가 변경되지 않았는지 확인

### 3. 내용 검증 (섹션별)
원문과 번역문을 섹션 단위로 대조하며 다음을 확인:

| 항목 | 기준 | 심각도 |
|------|------|--------|
| 의미 누락 | 원문의 문장/단락이 번역에서 빠졌는가 | 높음 |
| 의미 왜곡 | 원문과 다른 의미로 번역되었는가 | 높음 |
| 용어 불일치 | 같은 용어가 다르게 번역되었는가 | 중간 |
| 부자연스러움 | 번역체가 느껴지는 어색한 표현 | 중간 |
| 형식 깨짐 | 마크다운 구조가 손상되었는가 | 중간 |
| 코드 변경 | 코드 블록 내 코드가 수정되었는가 | 높음 |

### 4. 판정
- **PASS**: 즉시 사용 가능한 품질
- **FIX**: 수정 필요 — 수정 사항을 구체적으로 명시

### 5. 리뷰 리포트 작성

```markdown
## 리뷰: {filename}
- 판정: PASS | FIX
- 전체 품질: 상 | 중 | 하
- 구조 검증: 헤딩 {N}/{N}, 코드블록 {N}/{N}, 링크 {N}/{N}

### 수정 사항 (FIX인 경우)
1. **[높음]** 섹션: {섹션명}
   - 현재: {현재 번역}
   - 제안: {수정 제안}
   - 사유: {이유}

### 용어 일관성
- 확인된 용어 {N}개, 불일치 {N}개
- 불일치 목록: ...
```

## 판정 가이드라인

PASS 기준:
- 의미 누락/왜곡 없음
- 용어 불일치 없음
- 읽었을 때 자연스러운 글
- 형식 보존 완벽

FIX 기준 (하나라도 해당):
- 의미 누락 또는 왜곡 1건 이상
- 같은 용어의 번역이 2가지 이상
- 코드 블록 내 코드 변경
- 마크다운 구조 손상
